CTRL-PLANE

Demo Cue Card

— what to say when each button gets pushed
The pitch in one line: Cloud adoption created Cloud FinOps. Agentic AI needs AI FinOps. One gateway sits in front of every LLM call so you can route it cheaper, cache it, compress it, and cap runaway spend — without losing answer quality. It's a control plane, not a compressor.

1 The 4 levers (your opening explanation)

LeverPlain EnglishConcrete example
Router Picks the cheapest model that can still do the job. Escalates to a stronger model only if quality fails. "Fix this typo" → $0.30 small model. "Design this system" → $15 frontier model.
Semantic cache If the same request (by meaning, not exact text) was already answered, return the saved answer for ~$0. Two devs ask the same NPE question worded differently → 2nd is a cache hit.
Budget governor Enforces spend limits and kills runaway loops. Caps retries, trips a kill switch. Agent retries the same failing call forever → killed after 3 tries.
Payload compressor Shrinks the prompt/context before the call so you pay for fewer input tokens, without losing meaning. 50-page RAG context compressed (Headroom-style) before the model sees it.

2 What each button does

▶ Run agent — resolve JIRA-4821

the "good agent" / optimization story
  • One agent runs a real 16-step workflow to fix a bug ticket: triage → fetch code → summarize incidents → generate fix → review → write tests → changelog.
  • Watch each step route to the cheapest model that passes small mid frontier.
  • Repeated calls return CACHE HIT for free.
  • If a cheap model can't pass, it bumps up a tier: ESCALATED.
SAY: "Same work, ~93% cheaper, quality basically unchanged."

⚠ Run buggy agent (retry loop)

the "bad agent" / governance story
  • 5 normal steps, then the same "regenerate fix" call repeated 6 times — a broken agent stuck in a loop.
  • The governor counts identical calls. On the 4th repeat it fires KILLED — "loop detected".
  • The run stops mid-stream. No more tokens burned.
  • Also enforces a separate 800K-token-per-run budget cap.
SAY: "A runaway agent can't bankrupt you. The kill switch caps the blast radius automatically."

5 If someone pushes back (honest caveats)

This is a reference architecture / simulator, not production. Say so up front — it builds credibility.
  • Quality (Q) is modeled, not measured. In production you'd back it with real evals — graded tasks, human review, or an LLM judge — before claiming "quality maintained."
  • The cache matches near-literal repeats (exact or ≥0.85 token similarity). Real agents reword the same intent, so production needs embedding-based matching (cosine ≥ 0.95).
  • The kill thresholds are hardcoded (>3 identical calls, 800K tokens/run) and the buggy agent loops on an identical prompt — the easy case. Real runaways vary slightly, so you'd want fuzzy matching or cost-based circuit breakers.
  • The 31% cache rate is specific to this scripted ticket. Workflows with fewer repeats cache less.
  • State is in-memory today. Production swaps in Redis + Postgres, real provider calls (LiteLLM/Bedrock/OpenAI/Anthropic), and a learned router policy.
